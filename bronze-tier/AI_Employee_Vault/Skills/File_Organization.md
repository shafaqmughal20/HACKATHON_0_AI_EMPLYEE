# Skill: File Organization

## Purpose
Systematically organize files by type, date, and category to maintain a clean, efficient workspace.

## When to Use
- Messy directories with unorganized files
- New file drops requiring categorization
- Periodic cleanup and maintenance
- After completing projects
- When files are hard to find

## Prerequisites
- Read access to target directory
- Write access to create folders and move files
- Understanding of file types and purposes
- Backup created (if organizing important data)

## Procedure

### 1. Scan and Assess
```
1. List all files in target directory
2. Count total files
3. Identify file types present
4. Note any existing organization
5. Identify duplicates or temporary files
```

### 2. Plan Organization Structure
```
Based on file types, create logical structure:

By Type:
├── Documents/
├── Spreadsheets/
├── Images/
├── Data/
├── Code/
└── Archives/

By Date:
├── 2026-04/
├── 2026-03/
└── 2026-02/

By Project:
├── ProjectA/
├── ProjectB/
└── ProjectC/

Choose structure that best fits the content.
```

### 3. Create Folder Structure
```
1. Create main category folders
2. Create subcategories if needed
3. Verify folder permissions
4. Log folder creation
```

### 4. Categorize Files
```
For each file:
1. Identify file type and purpose
2. Determine appropriate category
3. Check for naming convention compliance
4. Note if renaming needed
```

### 5. Rename Files (if needed)
```
Standard naming convention:
YYYY-MM-DD_DescriptiveName.ext

Examples:
- report.pdf → 2026-04-05_QuarterlyReport.pdf
- data.csv → 2026-04-05_SalesData.csv
- image.jpg → 2026-04-05_ProductPhoto.jpg
```

### 6. Move Files
```
1. Move files to appropriate folders
2. Verify each move succeeded
3. Check file integrity after move
4. Log each file movement
```

### 7. Handle Special Cases
```
- Duplicates: Keep newest, archive or delete old
- Temporary files (.tmp, ~): Delete after verification
- Hidden files (.): Review before moving
- Large files: Note size, consider archiving
- Unknown types: Create "Other" category
```

### 8. Create Index
```
Create README.md in organized directory:

# Directory Index

## Structure
- Documents/ - PDF, Word, text files
- Spreadsheets/ - Excel, CSV files
- Images/ - Photos and graphics
- Data/ - JSON, XML, database files

## File Count
- Total files: X
- By category: ...

## Last Organized
- Date: YYYY-MM-DD
- By: AI Employee
```

### 9. Verify and Log
```
1. Count files in each category
2. Verify no files lost
3. Check all files accessible
4. Create detailed log entry
5. Update Dashboard
```

## Quality Checks
- [ ] All files accounted for (count matches)
- [ ] Files in logical categories
- [ ] Naming convention followed
- [ ] No duplicates (unless intentional)
- [ ] Folder structure makes sense
- [ ] Index/README created
- [ ] All moves logged
- [ ] Original directory cleaned up

## Common Pitfalls

### Pitfall 1: Over-categorization
**Problem**: Too many folders, hard to find things
**Solution**: Keep structure simple, 3-5 main categories max

### Pitfall 2: Losing Files
**Problem**: Files disappear during organization
**Solution**: Count files before and after, verify totals match

### Pitfall 3: Breaking Links
**Problem**: Moving files breaks references
**Solution**: Check for dependencies before moving

### Pitfall 4: Inconsistent Naming
**Problem**: Mixed naming conventions
**Solution**: Rename all files to standard format

```
    └── 2026-03-28_File3.md
```
